Implemented a first tak-core facade layer and adopted it in the root CLI binary:

1) New core facade module
- Added `crates/tak-core/src/facade.rs` and exported it via `crates/tak-core/src/lib.rs`.
- Introduced `RepoContext` (repo discovery/open + task-id resolution helpers).
- Added reusable task-id resolver helpers:
  - `resolve_task_id_arg`
  - `resolve_optional_task_id_arg`
  - `resolve_task_id_args`
- Added coordination verbosity helpers moved from CLI main:
  - `resolve_effective_coordination_verbosity`
  - `apply_coordination_verbosity_label`
  - `maybe_add_verbosity_tag`
  - `task_assignee_for_verbosity`
- Added output/error facade types/hooks:
  - `ProcessOutput`
  - `render_error_message`
  - `execute_with_context`
  - `execute_with_discovered_repo`

2) Root CLI adoption
- Updated `src/main.rs` helper functions to delegate to `tak::facade::*` instead of local repo/verbosity logic.
- Updated top-level error rendering in `main()` to use `tak::facade::render_error_message`.
- Adjusted tree dispatch to map existing `--pending` bool to core `TreeScope` enum (`Pending`/`All`).

3) Validation
- `cargo test -p tak-core facade::tests`
- `cargo test -p tak --no-run`
- `cargo test -p tak-cli --no-run`
- `cargo test --workspace --no-run`

All checks pass after `cargo fmt`.
